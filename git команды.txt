git branch — она показывает список веток, с которыми мы работаем в проекте, и выделяет текущую.
git checkout — перемещение по веткам. После git checkout надо указать название нужной ветки. Например, git checkout master — переход в ветку master.
git status — показывает текущее состояние в рабочей ветке, а именно список с названиями изменённых файлов, если они есть, и указывает на те, которые ожидают записи и сохранения (обычно они выделены красным цветом).
git add — сохраняет изменения. Или "индексирует", то есть, сообщает Git, что именно мы собираемся сохранять. git add имя-файла. git add -A — чтобы не перечислять каждый файл, в который были внесены изменения. git add . — точка означает текущую папку, значит эта команда проиндексирует всю папку.
git commit -m "ваше сообщение" — после того, как все файлы проиндексированы, можно создать коммит, то есть зафиксировать все сохранённые изменения и дать им название.
git push origin module1-task1 (имя ветки, оно мб любым) — отправка локальных изменений (коммитов) в удалённый репозиторий, команда push (толкать).
